## 6. 集成修复记录

### 6.1 首页统计数据键名不匹配

问题：`1_首页.py` 中调用 `literature_processor.get_stats()` 和 `recipe_extractor.get_stats()` 后，使用了错误的字典键访问统计数据。

原代码使用 `lit_stats.get("literature", {}).get("total", 0)` 和 `recipe_stats.get("recipes", {}).get("total", 0)`，但 `get_stats()` 直接返回 `{"total": N, ...}` 结构。

修复：将键访问改为 `lit_stats.get("total", 0)` 和 `recipe_stats.get("total", 0)`。

### 6.2 教师分析面板重复 set_page_config

问题：`8_教师分析面板.py` 中的 `main()` 函数调用了 `st.set_page_config()`，但主入口 `app.py` 已设置页面配置。在 Streamlit 多页面应用中，页面级 `set_page_config` 与主应用冲突。

修复：移除 `8_教师分析面板.py` 中的 `st.set_page_config()` 调用。

## 8. 安全性测试

### 8.1 API 认证

- 所有 API 端点均需 X-API-Key 请求头
- 无 Key 或无效 Key 返回 401
- CORS 仅允许 localhost 端口

### 8.2 SSRF 防护

- Ollama URL 设置接口仅允许 localhost/127.x/10.x/172.16-31.x/192.168.x
- 外部地址被拒绝

### 8.3 输入验证

- Pydantic 模型校验所有请求参数
- query 字段长度限制 1-5000 字符
- model 名称长度限制 1-200 字符
- feedback rating 范围 0-5

### 8.4 高风险实验检测

- 浓硫酸、液氮、氢氟酸等关键词触发高风险标记
- 自动推荐教师复核清单
- 教学版声明和安全边界在所有相关页面展示
