# SoftMatterGPT 软件说明书

## 1. 系统概述

SoftMatterGPT 是面向软物质课程与实验教学的 AI 教学工作台。系统基于公开大语言模型（Ollama），采用检索增强生成（RAG）架构，为师生提供课程知识问答、文献检索、实验配方查询、实验方案生成、表征结果辅助分析、材料体系对比以及教学数据分析等功能。

系统采用前后端分离架构：前端为 Streamlit 多页面应用（9 个功能页面），后端为 FastAPI RESTful API 服务（13 个业务 API 端点，15 个独立路径）。数据层基于 CSV 文件和 JSON 配置文件，检索层采用 scikit-learn 的 TF-IDF 向量化实现文本匹配。

## 2. 核心模块说明

### 2.1 全局配置（config.py）

集中管理所有可调参数，前端和后端共享同一配置源。主要配置项包括：

- **项目路径**：PROJECT_ROOT、DATA_DIR、LOGS_DIR、ASSETS_DIR、DOCS_DIR
- **数据文件路径**：课程知识库、文献记录、实验配方、问答历史、反馈记录、设置文件
- **Ollama 配置**：服务地址（默认 http://localhost:11434）、默认模型（默认 qwen2.5:7b）
- **RAG 参数**：Top-K 返回条数（默认 5）、相似度阈值（默认 0.3）
- **服务配置**：API 监听地址与端口、Streamlit 端口
- **认证配置**：API Key，支持环境变量设置，未设置时自动生成随机密钥
- **安全配置**：高风险关键词列表（强酸、浓硫酸、液氮等）、安全免责声明文本

敏感信息通过环境变量读取，不硬编码在代码中。

### 2.2 LLM 客户端（core/llm_client.py）

封装与 Ollama 服务的交互逻辑，主要功能：

- **连接检测**：通过 `/api/tags` 接口检测 Ollama 是否在线
- **模型列表获取**：返回本地已拉取的模型列表
- **当前模型管理**：支持获取和设置当前使用模型，持久化到 `settings.json`
- **模型测试**：发送简单 prompt 验证模型是否正常响应
- **文本生成**：调用 Ollama `/api/generate` 接口生成文本

离线行为：当 Ollama 服务不可用时，所有方法返回友好的错误信息，不抛出异常、不崩溃。前端和后端均能正常启动和运行。

### 2.3 知识检索器（core/retriever.py）

基于 TF-IDF 向量化的课程知识库检索模块，主要功能：

- **索引构建**：加载 `softmatter_knowledge.csv`，构建 TF-IDF 索引
- **中文优化**：通过词面重合度补充机制改善中文短查询的召回率
- **别名扩展**：内置软物质领域别名表（如 CMC/临界胶束浓度、G'/储能模量），自动扩展查询
- **相似度阈值**：低于 0.3 的结果自动过滤，减少噪声
- **跨库检索（search_all）**：合并知识库、文献库、配方库三库结果，按分数统一排序

### 2.4 文献处理器（core/literature_processor.py）

负责文献记录的管理和检索，主要功能：

- **记录检索**：支持子串匹配，可按关键词、年份、材料体系等维度筛选
- **数据导入**：支持 CSV、TXT、Markdown 三种格式导入，CSV 导入支持自动去重
- **统计信息**：返回记录总数、按数据来源分类统计

文献记录包含字段：id、title、year、source、material_system、method、characterization、result_summary、keywords、data_source。

### 2.5 实验配方抽取器（core/recipe_extractor.py）

负责实验配方的管理和检索，主要功能：

- **配方检索**：支持子串匹配，可按材料、安全等级等维度筛选
- **文本抽取**：从自然语言描述中通过正则模式匹配提取配方字段
- **安全等级推断**：基于关键词规则自动推断配方的安全等级

配方记录包含字段：id、material、concentration、process、temperature、time、characterization、result、safety_level、source、is_example。

### 2.6 安全审核器（core/safety_checker.py）

对实验方案和文本内容进行安全风险评估，主要功能：

- **风险关键词检测**：三级风险分类（高风险：强酸、浓硫酸、液氮等；中风险：有机溶剂、加热等；低风险：常温、水溶液等）
- **文本安全审核**：输入文本，输出风险等级和具体风险项
- **配方安全审核**：针对实验配方进行专项安全评估
- **教师复核清单生成**：高风险项自动生成教师复核建议清单

### 2.7 Prompt 模板管理器（core/prompt_templates.py）

集中管理各类任务的 Prompt 模板，确保 LLM 输出符合教学场景要求。模板类型包括：

- **通用问答模板**：要求模型优先依据证据、不编造内容、适合本科生理解、涉及实验必须提醒安全
- **实验方案模板**：强制包含"教学版实验方案框架"声明，明确不可作为真实 SOP
- **文献抽取模板**：结构化抽取材料体系、实验方法、表征结果等字段
- **表征分析模板**：要求只根据输入现象解释，不下确定结论，输出可能原因和验证建议

### 2.8 表征结果分析器（core/characterization.py）

基于规则匹配的表征结果辅助分析模块，主要功能：

- **规则库**：内置 9 条分析规则，覆盖剪切变稀/增稠、储能模量与损耗模量关系、多孔结构、相分离、胶束形成、FTIR 峰位、LCST 行为、Pickering 乳液等常见表征现象
- **文本分析**：输入表征结果的文字描述，匹配相关规则并结合知识库证据
- **置信度计算**：根据证据数量和规则匹配情况动态计算置信度
- **输出结构**：观察到的现象、可能对应结构、可能工艺原因、建议补充表征、风险与不确定性

### 2.9 问答日志记录器（core/qa_logger.py）

记录用户问答和反馈数据，主要功能：

- **QA 记录写入**：记录问题、回答摘要、证据数量、时间戳等信息到 `qa_history.csv`
- **反馈记录写入**：记录评分（0-5）、评论、时间戳到 `feedback.csv`
- **关键词提取**：基于领域术语词典和正则分词提取关键词，无外部 NLP 依赖

### 2.10 教师统计分析器（core/analytics.py）

对问答日志和反馈数据进行统计分析，为教师提供教学参考，主要功能：

- **问题计数**：统计总问答次数和今日提问次数
- **反馈分布**：统计正面、中性、不满意反馈数量和满意率
- **高频关键词**：领域术语优先提取，输出 Top-N 关键词及频次
- **每日趋势**：按日期统计提问数量
- **弱点总结**：基于低置信度和证据不足的问答生成薄弱知识点分析
- **复习建议**：根据高频问题和薄弱点生成课堂复习建议

### 2.11 数据加载器（core/data_loader.py）

负责从 CSV 和 JSON 文件加载系统所需的所有数据，主要功能：

- **课程知识库加载**：读取 `softmatter_knowledge.csv`，返回结构化数据列表
- **文献记录加载**：读取 `literature_records.csv`，支持按数据来源过滤
- **实验配方加载**：读取 `experiment_recipes.csv`，支持按示例标记过滤
- **测试问题加载**：读取 `sample_questions.csv`，供演示和测试使用
- **设置文件加载**：读取 `settings.json`，管理模型选择和 Ollama 地址等配置
- **容错处理**：文件不存在或格式异常时返回空列表，不抛出异常

### 2.12 前端架构说明

Streamlit 多页面应用采用主入口 + 子页面的结构：

- **主入口（app.py）**：设置全局页面配置（标题、图标、布局），调用各页面的 `main()` 函数
- **页面路由**：通过文件名前缀的数字编号实现页面排序，Streamlit 自动识别 `pages/` 目录下的文件
- **状态管理**：通过 `st.session_state` 管理跨页面的共享状态（如当前模型名称、Ollama 在线状态）
- **API 通信**：所有页面通过 HTTP 请求调用后端 API，使用 `requests` 库，统一携带 `X-API-Key` 请求头
- **离线降级**：每个页面独立处理 API 调用失败的情况，显示友好的错误提示，不影响其他功能使用

## 3. 前端页面功能说明

### 3.1 首页（1_首页.py）

系统总览页面，展示以下信息：

- **项目标识**：SoftMatterGPT 项目名称和一句话定位
- **状态数据卡片**：当前模型名称、Ollama 在线/离线状态、课程知识库条数、文献记录条数、实验配方条数、今日提问次数、累计问答次数、反馈满意率
- **系统架构图**：展示用户浏览器 -> Streamlit -> FastAPI -> 核心模块 -> 数据层 -> Ollama 的完整架构
- **安全声明**：页面底部展示系统安全免责说明

所有数据从真实数据源读取，不伪造。

### 3.2 软物质智能问答（2_软物质智能问答.py）

基于 RAG 的课程知识问答页面：

- **输入**：用户输入软物质相关问题
- **检索**：系统从知识库、文献库、配方库三库检索相关证据
- **生成**：调用 LLM 基于检索证据生成回答
- **输出格式**：简要结论、详细解释、例子说明、容易混淆点、参考证据、学习建议、置信度
- **证据卡片**：展示 Top-K 条证据，包含标题、分类、来源、相似度、内容摘要
- **反馈功能**：用户可对回答进行满意/一般/不满意评价
- **离线处理**：LLM 不可用时，仍展示已检索的证据，并显示错误提示

### 3.3 文献知识检索（3_文献知识检索.py）

结构化文献记录检索页面：

- **检索功能**：支持关键词搜索，可按年份、材料体系等维度筛选
- **记录展示**：展示论文标题、年份、来源、材料体系、实验方法、表征方式、结果摘要
- **数据导入**：支持上传 CSV、TXT、Markdown 格式文件导入文献记录
- **统计信息**：展示当前文献记录总数和数据来源分布

### 3.4 实验配方库（4_实验配方库.py）

实验配方查询与管理页面：

- **配方检索**：支持关键词搜索，可按材料、安全等级筛选
- **配方展示**：展示材料、浓度、工艺条件、温度、时间、表征结果
- **安全等级**：每个配方标注安全等级（高/中/低）
- **教师复核**：高风险配方自动显示教师复核提示
- **示例标注**：教学示例数据明确标注，不伪装为真实文献数据

### 3.5 实验方案助手（5_实验方案助手.py）

教学版实验方案生成页面：

- **输入字段**：研究目标
- **生成内容**：实验目标、设计思路、参考配方、材料体系、变量设计、表征方法、预期现象、风险提醒、教师复核清单
- **教学声明**：输出明确标注为"教学版实验方案框架"，不可作为真实可执行 SOP
- **安全审核**：高风险试剂和条件自动标记教师审核要求

### 3.6 表征结果分析（6_表征结果分析.py）

表征结果辅助分析页面：

- **文字分析**：用户输入显微图或流变曲线的文字描述，系统基于规则库和知识库进行分析
- **图片上传**：提供图片上传入口（当前为预留接口，未接入多模态模型）
- **分析输出**：观察到的现象、可能对应结构、可能工艺原因、建议补充表征、风险与不确定性
- **免责声明**：明确标注分析结果仅作为教学解释，不作为最终实验结论

### 3.7 材料体系对比（7_材料体系对比.py）

材料体系对比分析页面：

- **预设对比主题**：物理交联水凝胶 vs 化学交联水凝胶、表面活性剂乳液 vs Pickering 乳液、牛顿流体 vs 非牛顿流体等
- **自定义对比**：用户可输入自定义材料进行对比
- **输出格式**：Markdown 对比表格，展示各项性能指标对比结果
- **证据来源**：对比结论附带证据来源

### 3.8 教师分析面板（8_教师分析面板.py）

面向教师的教学数据分析页面：

- **最近提问记录**：展示近期学生提问列表
- **高频关键词柱状图**：展示学生高频关注的知识点
- **反馈分布饼图**：展示学生对回答的满意度分布
- **每日提问趋势**：折线图展示每日提问数量变化
- **弱点总结**：基于低置信度问答分析学生薄弱知识点
- **复习建议**：根据高频问题和薄弱点自动生成课堂复习建议

### 3.9 系统设置（9_系统设置.py）

系统配置管理页面：

- **Ollama 连接检测**：一键检测 Ollama 服务是否在线
- **服务地址配置**：支持修改 Ollama 服务地址（含 SSRF 防护，仅允许内网地址）
- **模型列表刷新**：从 Ollama 获取本地已拉取的模型列表
- **模型选择**：下拉框选择当前使用的模型
- **模型测试**：发送测试 prompt 验证模型是否正常工作

## 4. 后端 API 说明

### 4.1 认证机制

所有 API 端点均需在请求头中携带 `X-API-Key`。API Key 通过环境变量 `SOFTMATTER_API_KEY` 配置，未设置时系统自动生成随机密钥。

### 4.2 输入验证

所有请求参数经 Pydantic 模型校验：
- query 字段：1-5000 字符
- model 名称：1-200 字符
- feedback rating：0-5 整数
- description 字段：1-10000 字符

### 4.3 CORS 策略

仅允许来自 localhost 的跨域请求，不对外网开放。

### 4.4 SSRF 防护

Ollama 地址设置接口严格校验 URL，仅允许 localhost、127.x.x.x、10.x.x.x、172.16-31.x.x、192.168.x.x 等内网地址。

## 5. 数据层说明

### 5.1 数据文件

| 文件 | 说明 | 当前条数 |
|------|------|----------|
| softmatter_knowledge.csv | 课程知识库 | 60 条 |
| literature_records.csv | 文献记录 | 27 条 |
| experiment_recipes.csv | 实验配方 | 30 条 |

| feedback.csv | 反馈记录 | 6 条 |

### 5.2 数据真实性

- 课程知识库为人工编写的教学知识条目

- 实验配方为教学示例数据（标注 is_example=true）
- 系统支持后续导入真实数据，当前数据状态在 settings.json 中如实标注

## 6. 系统运行要求

### 6.1 硬件要求

- CPU：双核及以上
- 内存 >= 4 GB
- 磁盘空间 >= 500 MB（不含 Ollama 模型文件）

### 6.2 软件要求

- 操作系统：Windows 10/11、macOS 12+、Ubuntu 20.04+
- Python >= 3.9（推荐 3.11）
- pip 包管理器
- 如需 LLM 功能：Ollama 已安装并运行，至少拉取一个模型（推荐 qwen2.5:7b）
- 如仅使用检索和规则分析功能：无需 Ollama，系统可独立运行

### 6.3 网络要求

- 系统设计为单机本地部署，无需外网访问
- 前后端通信走 localhost，默认端口 8000（API）和 8501（Streamlit）
- Ollama 服务默认监听 11434 端口
- 安装 Ollama 和拉取模型时需要网络连接，运行时无需外网

### 6.4 部署步骤概要

1. 克隆项目代码并安装 Python 依赖（`pip install -r requirements.txt`）
2. 可选安装 Ollama 并拉取推荐模型（`ollama pull qwen2.5:7b`）
3. 可选配置环境变量（API Key、Ollama 地址等）
4. 启动后端服务（`python api_server.py`）
5. 启动前端页面（`streamlit run app.py`）
6. 在浏览器中访问 `http://localhost:8501` 即可使用
